# Unknown Explainability Review Pack

## Purpose
Use this pack to collect operator feedback on unknown-confidence wording with concrete persisted unknown-result examples.

Scope guardrails for this review:
- Explainability wording interpretation only.
- No verdict/classifier/readiness authority changes.
- Unknown is not equivalent to allowed.

## Sample Sources
- Source file scanned: `tests/routes/test_api_routes.py` (`api_routes.ResultRow` fixtures with `verdict="unknown"`).
- Sample type: persisted `result.report_json` row fixtures in route tests.
- Synthetic fallback samples: none.

## Review Rubric
For each sample, score each item 1-5 and add notes:
- Is the dominant cause understandable?
- Is the wording actionable for first-hop triage?
- Does wording avoid implying `allowed`?
- Is wording too technical or too vague?
